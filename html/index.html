CI/CD Pipeline Proof of Concept

Overview
This website describes the unseen infrastructure and automation that continously takes some lines of code and turns it into a working product. This project had two goals:
  1. Get hands-on experience with some really cool enterprise-grade technologies
  2. Be entirely cost-free
The combination of these goals challenged me to better understand the services and to seek out creative solutions.

Infrastructure Components
Content in-progress This (mostly) HTML website is being served from a docker container running Nginx. The container is hosted I'm always curious how organizations decide between AWS, Azure, and GCP, so in good spirit, this is why I chose Azure: I knew I wanted to serve the webside from a container and Microsoft offers a free tier for App Service. They also offer a free tier for Azure DevOps. The free tiers also seemed like they had workable constraints. AWS has an equivalent service called Fargate; however, AWS' "free tier" has a different model than Microsoft's that only lasts for a year and that felt like it would be cheating on goal #2. I also know how quickly a year really passes. GCP recently announced Cloud Run which has a always free tier similar to Microsoft's; however, if I went with Cloud Run, then I'd need to source my pipeline tool serperately. So, to keep costs actually at zero and to take advantage of the same-provider synergy I chose Azure. The container is hosted in an Azure App Service.
